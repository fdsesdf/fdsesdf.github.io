C++ 大型流媒体项目学习笔记

C++ 大型流媒体项目学习笔记

一、开发环境安装与工程搭建

1. Linux 开发环境

后台服务通常部署在 Linux 系统上，因此开发阶段应尽量在 Linux 环境中完成编译与测试。

常见方式包括：

  • 物理机安装 Linux
  • 虚拟机（VMware / VirtualBox）
  • 远程 Linux 服务器

开发环境与线上运行环境保持一致，可以有效避免兼容性问题，降低部署风险。

2. Samba 与终端工具

在跨平台开发中，合理使用工具可以显著提升效率：

  • Samba：实现 Windows 与 Linux 之间的文件共享
  • 终端工具：用于远程登录与命令行操作（SSH）

后续流媒体开发中常用工具包括：

  • FFmpeg
  • OBS
  • VLC

3. VS Code 远程开发

VS Code 支持 Remote-SSH 插件，实现本地编辑、远程编译与调试，可直接操作 Linux 服务器代码，适合后台服务开发。

4. CMake 构建工程

CMake 是一种跨平台构建工具，用于生成和管理 Makefile。
相比手写 Makefile，CMake 更适合中大型项目，工程结构清晰、维护成本低。

CMake 是一种跨平台构建工具，用于生成 Makefile 或 Ninja 文件。

优势：

  • 语法简洁
  • 工程结构清晰
  • 易于维护和扩展
  • 适合中大型项目

相比手写 Makefile，CMake 更适合长期维护的流媒体系统工程。

1. 时间相关函数

2. 字符串与路径处理函数

三、单例设计模式

1. 设计目标

  • 保证一个类在程序生命周期中只有一个实例
  • 提供全局访问点

2. 实现要点

  • 构造函数私有化
  • 多线程安全（pthread_once）
  • 禁止拷贝与移动
  • 遵循 C++ 三 / 五法则

四、定时任务系统

1. 定时任务结构

  • when
  • interval
  • callback

2. 定时任务管理器功能

  • 添加、删除定时任务
  • 检测任务是否到期
  • 执行到期任务
  • 全局唯一（单例）

3. 设计特点

  • 任务数量少，直接遍历
  • 不需要重排任务
  • 使用互斥锁保证线程安全

总结

本文整理了 C++ 流媒体项目的基础工程与核心设计，为后续音视频系统开发奠定了扎实基础。
